SQL Interview Questions

1. What is SQL?

SQL (Structured Query Language) is a standard language for accessing and manipulating databases.

2. What is the difference between DELETE and TRUNCATE?

- DELETE: Removes specific rows from a table and can have a WHERE clause. Logs individual row deletions.
- TRUNCATE: Removes all rows from a table and resets identity. Faster but not logged row by row.

21. What are SQL constraints?

Constraints are rules enforced on table columns to maintain data integrity:

  • NOT NULL: Ensures column cannot have NULL values.
  • UNIQUE: Ensures all values in a column are unique.
  • CHECK: Ensures values meet a specific condition.
  • DEFAULT: Sets a default value for a column.
  • FOREIGN KEY: Establishes a relationship between two tables.

22. How do you use the CASE statement in SQL?

The CASE statement is used for conditional logic within SQL queries:


SELECT employee_id, 
       CASE 
           WHEN salary > 5000 THEN 'High'
           WHEN salary BETWEEN 3000 AND 5000 THEN 'Medium'
           ELSE 'Low'
       END AS Salary_Level
FROM employees;

23. Explain the difference between OLTP and OLAP.

- OLTP (Online Transaction Processing): Handles day-to-day database operations like insert, update, delete.
- OLAP (Online Analytical Processing): Used for analytical queries and data aggregation.

24. How do you create a composite key?

A composite key is created using multiple columns to uniquely identify a row in a table:


CREATE TABLE orders (
    order_id INT,
    product_id INT,
    quantity INT,
    PRIMARY KEY (order_id, product_id)
);

25. What is the purpose of the COALESCE function?

The COALESCE function returns the first non-NULL value from a list of expressions:


SELECT COALESCE(NULL, NULL, 'SQL', 'Backup') AS result;
-- Output: SQL

26. How do you use window functions in SQL?

Window functions perform calculations across a set of rows related to the current row:


SELECT employee_id, 
       salary, 
       RANK() OVER (ORDER BY salary DESC) AS rank
FROM employees;

27. Explain the difference between a temporary table and a CTE.

  • Temporary Table: Created explicitly and persists until dropped or the session ends.
  • CTE (Common Table Expression): A temporary result set used within a single query scope.

28. How do you create a recursive query using a CTE?

Recursive CTEs allow hierarchical data queries:


WITH cte_name AS (
    SELECT column1, column2
    FROM table
    WHERE condition
    UNION ALL
    SELECT column1, column2
    FROM table
    JOIN cte_name ON condition
)
SELECT * FROM cte_name;
